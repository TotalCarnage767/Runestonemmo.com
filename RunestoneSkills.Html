RunestoneSkills

Progression for Gatherers, Crafters, and Wanderers — Forged to Fit the Core.

RunestoneSkills is the official progression addon for the RunestoneMMO ecosystem. It adds MMO-style skill leveling for gathering, crafting, and exploration — while integrating cleanly into the same APIs, XP curve rules, placeholders, and UI/scoreboard layer used by RunestoneCore.

Minecraft Java 1.21.11 • Paper • JDK 21 • Modular Addon Architecture

Compatibility

Platform

  • Server: Paper
  • Minecraft: 1.21.11
  • Java: 21

Dependencies

  • Required: RunestoneCore
  • Optional: Oraxen (item support templates)
  • Optional: PlaceholderAPI (for wider placeholder coverage; core placeholders still work)

Design Goals

  • Config-driven progression (YAML-first tuning)
  • Anti-exploit safeguards by default
  • Performance-minded (rate limits, lightweight checks)
  • Clean Core integration (XP service + skill registry + scoreboard placeholders)

What RunestoneSkills Adds

Skill Progression (Modular)

  • Multiple skills with separate configs and tuning
  • Unified level rules shared with RunestoneCore systems
  • XP gain events and level-up behavior routed through Core APIs

Anti-Exploit Safeguards

  • Placed block tracking (player-placed = 0 XP where applicable)
  • Cooldowns / minimum intervals (per skill, tuneable)
  • Optional rate limits (per-player XP caps)
  • Correct-tool validation (pickaxe/axe/etc.) where applicable
  • Validation checks (maturity, natural-only, right-click harvest rules)

UI & Display

  • /skills shows all registered skills
  • Scoreboard placeholder support for skill levels and progress
  • Designed to coexist with RunestoneCore’s scoreboard system and layouts

Configuration

RunestoneSkills is configured through separate skill YAML files. Each skill can be enabled/disabled and tuned independently. Files generate into plugins/RunestoneSkills/.

Skill Config Files

  • mining.yml
  • forestry.yml
  • fishing.yml
  • hunting.yml
  • farming.yml
  • cooking.yml
  • smithing.yml
  • exploration.yml
  • foraging.yml

Tuneables You Control

  • XP values (per material, per source)
  • Max level per skill
  • Cooldowns / rate limits
  • Tool requirements and validation rules
  • Optional Oraxen XP mappings (template sections)

Balancing Notes

  • Configs are intended to be re-tuned frequently during development
  • Deleting a live skill YAML regenerates defaults from the jar (safe during early balancing)
  • Long-term: missing-key merges can be added to preserve edits while gaining new keys

Skills Included

Each skill is implemented as a focused subsystem: it listens for relevant events, validates actions, applies anti-exploit rules, and grants XP through the unified progression API surface.

Mining

  • XP for mining blocks with the correct tool (pickaxe validation)
  • Config-driven XP tables per block/material
  • Placed-block protection (anti-farm)
  • Optional override generation (tag-based) for coverage, plus manual tuning

Forestry

  • XP for cutting natural trees only
  • Axe validation + natural-only rules
  • Placed-log tracking to prevent farm loops
  • Config-driven XP per wood/log type

Fishing

  • Catch-based XP (configurable tables)
  • Anti-abuse intervals and filters
  • Tuneable progression pacing

Hunting

  • XP for passive mob kills (config-driven per entity)
  • Anti-abuse filters (spawn/circumstance rules, tuneable)
  • Designed for clean integration with economy and drops

Farming

  • XP for harvesting crops and plants (maturity/validity checks)
  • Config-driven XP per crop
  • Optional crop-trample prevention (config-driven)

Cooking

  • XP for cooking/crafting food outputs (config-driven)
  • Supports common cooking sources and recipes (tuneable)
  • Anti-spam controls (intervals/rate limits)

Smithing

  • XP for crafting weapons/tools/armor (config-driven lists)
  • XP for smithing upgrades (smithing table outputs)
  • XP for relevant smelting outputs (ingots/scrap, tuneable)
  • Oraxen template sections for weapons/tools/armor/smelting categories

Exploration

  • Background travel XP (tuneable)
  • Biome discovery XP (tuneable per biome)
  • Landmarks framework for future world points of interest
  • Anti-exploit rules to prevent macro loops and abuse

Foraging

  • Wild vegetation gathering (mushrooms, flowers, grass/ferns, vines, moss/lichen, etc.)
  • Berry harvesting via right-click rules (no breaking the plant)
  • Natural-only + placed protection for anti-farm
  • Config-driven XP per vegetation type

Scoreboard & Placeholder Support

RunestoneSkills registers skills into the Core skill registry. This allows commands and scoreboards to display consistent level and progress information using stable placeholders.

Core Skill Placeholders

  • %skills_<id>_level%
  • %skills_<id>_progress%
  • %skills_<id>_xp_into_level%
  • %skills_<id>_xp_to_next%

Example IDs: mining, forestry, fishing, hunting, farming, cooking, smithing, exploration, foraging.

Command Display

  • /skills displays all registered skills and their progression
  • Designed to keep output consistent across combat and non-combat skills

UI Philosophy

  • Scoreboards are best for a compact overview
  • GUI skill menus are ideal for full detail (XP totals, bars, categories, icons)
  • RunestoneSkills is built so UI upgrades can be layered on safely

Commands

RunestoneSkills is designed to integrate through Core’s command and display layer. Available commands may vary depending on server configuration and enabled subsystems.

Player Commands

  • /skills — show skill levels and progress
  • /rskills — alias (if enabled via Core command aliasing)

Admin / Tuning Workflow

  • Skill values are tuned through the YAML files in plugins/RunestoneSkills/
  • Restart recommended during active tuning (safe lifecycle)
  • Optional plugin reload command may be enabled depending on build/version

Performance & Safety

Performance Minded

  • Validation-first checks to avoid expensive work
  • Optional rate limit caps for XP gain
  • Config-driven intervals to prevent spam events

Exploit Resistance

  • Placed tracking for blocks that should not grant XP when placed
  • Natural-only checks in gathering skills where needed
  • Tool checks for mining/forestry

Modular Expansion

  • Designed so future modules can register additional skills safely
  • Skill IDs remain stable for placeholders and UI integrations
  • Progression logic stays consistent across the ecosystem
