What RunestoneSkills Is

RunestoneSkills is the progression layer of the RunestoneMMO ecosystem. It is designed to make professions, gathering, crafting, and exploration feel like part of a larger MMO framework instead of isolated mechanics.

The module registers its skills into RunestoneCore so players can view them through the shared skills display, while server owners tune each skill independently through dedicated configuration files.

Compatibility

Platform

  • Server: Paper
  • Minecraft: 1.21.11
  • Java: 21

Dependencies

  • Required: RunestoneCore
  • Optional: Oraxen
  • Built to plug into Core’s skill registry, XP handling, and UI flow

Design Goals

  • Config-driven tuning per skill
  • Anti-exploit safeguards by default
  • Performance-minded event handling
  • Clean integration with Core instead of duplicating systems

What RunestoneSkills Adds

Skill Progression

  • Multiple skills with independent configs and progression tuning
  • Shared level display flow through the broader RunestoneMMO ecosystem
  • Unified registration into the Core skills registry
  • Designed for long-term expansion without fragmenting progression systems

Anti-Exploit Protection

  • Placed block tracking where applicable
  • Natural-only validation in skills that require it
  • Correct-tool checks for skill actions such as mining and forestry
  • Cooldowns, rate limits, and validation-first behavior to reduce abuse

Display & Integration

  • /skills displays registered skills through the shared Core-facing layer
  • Stable skill IDs for UI, placeholders, and future expansion
  • Built to coexist cleanly with Core scoreboards and future skills UI upgrades

Configuration

RunestoneSkills is built around per-skill configuration. Each major skill has its own YAML file, allowing balancing and tuning without forcing all progression systems into one config.

Skill Config Files

  • mining.yml
  • forestry.yml
  • fishing.yml
  • hunting.yml
  • farming.yml
  • cooking.yml
  • exploration.yml
  • foraging.yml
  • smithing.yml

Tuneables You Control

  • XP values by material, action, or source
  • Per-skill max levels
  • Tool requirements and validation rules
  • Intervals, rate limits, and exploit-prevention behavior
  • Skill enable / disable state by config

Admin Workflow

  • Configs generate into plugins/RunestoneSkills/
  • Early balancing can be handled by editing individual skill files
  • Prefer proper restarts for major tuning changes
  • Admin control is exposed through /runestoneskills reload

Skills Included

Each skill is implemented as its own focused subsystem: it listens for relevant actions, validates them, applies anti-exploit rules, and grants XP through the shared progression flow.

Mining

  • XP for valid mined materials with correct-tool support
  • Config-driven XP mappings
  • Placed-block anti-farm protections
  • Designed for controllable progression pacing

Forestry

  • Tree-cutting progression with natural-tree protections
  • Axe validation and anti-abuse logic
  • Placed-log protection and traversal safeguards
  • Ongoing balancing and edge-case refinement for awkward tree structures

Fishing

  • Catch-based XP progression
  • Config-driven pacing and reward flow
  • Anti-abuse intervals and filters

Hunting

  • Passive-mob-focused progression flow
  • Entity-based tuning and filtering
  • Designed to stay configurable and server-balance friendly

Farming

  • Crop and plant harvesting progression
  • Maturity and validity checks
  • Config-driven harvest tuning

Cooking

  • Food crafting and cooking progression
  • Config-driven outputs and pacing
  • Anti-spam timing controls

Exploration

  • Background travel XP
  • Biome discovery progression
  • Designed to support discovery-oriented world play
  • Built with anti-loop and anti-macro intent

Foraging

  • Wild vegetation and natural-gathering progression
  • Berry and plant interaction rules
  • Natural-only protections to limit farming abuse
  • Config-driven plant and vegetation XP tuning

Smithing

  • Weapons, tools, armor, upgrades, and related progression hooks
  • Crafting and smithing-table support
  • Smelting-related progression tuning where configured
  • Built for continued expansion alongside RunestoneMMO item systems

Commands & Skill Display

RunestoneSkills integrates into the Core display layer rather than trying to duplicate a separate player-facing command system.

Player-Facing Display

  • /skills shows registered skills and progression through the shared RunestoneMMO display flow
  • Enabled skills appear through Core once registered
  • Skill presentation is designed to remain consistent across the ecosystem

Plugin Admin Command

  • /runestoneskills reload
  • Permission: runestone.skills.admin
  • Used for administration and configuration workflow

UI Direction

  • Compact overview through shared displays
  • Room for future expanded GUI skill views
  • Stable skill IDs for future UI, placeholder, and scoreboard growth

Performance & Safety

Performance Minded

  • Validation-first event handling
  • Skill-specific pacing and timing controls
  • Built to avoid unnecessary heavy processing

Exploit Resistance

  • Placed-block protections where needed
  • Natural-only logic for appropriate gathering skills
  • Tool checks and action validation for cleaner progression

Modular Expansion

  • Built so more skills can be added cleanly over time
  • Stable registration model for future modules
  • Designed to grow with the wider RunestoneMMO project

Built to Work With RunestoneCore

RunestoneSkills is not meant to stand alone — it is meant to extend the Core. Economy, scoreboards, UI, future questing, future professions, and long-term progression all become stronger when skills live on the same shared foundation.
