Current State of Development

RunestoneCore is a powerful engine with many systems already implemented — economy, auctions, shops, moderation, combat scaffolding, attributes, daily rewards, world rules, scoreboards, teleports, claims, towns/territories, and more. The MMO layer is actively expanding through modular progression and content systems.

Fully Implemented

  • Multi-currency economy (wallets, bank, taxes, exchange, treasury, ledger)
  • Auctions with escrow, expiration, and offline delivery
  • Player shops (core flows and persistence)
  • Daily rewards with streaks and calendar logic
  • Combat XP and damage engine (crit, armor, scaling, floors)
  • Mob attribute application on spawn
  • Attribute platform for players and mobs
  • Moderation suite (punishments, reports, notes, jail, maintenance, audit trail)
  • Chat control and staff chat
  • Scoreboard system with layouts and conditions
  • Menu framework + cleanup/lifecycle
  • Cooldown system (disk-backed, config-driven)
  • Mob health bars
  • World rules and regions (rectangular zones)
  • Sleep system
  • Teleport suite: homes, spawn, back, TPA
  • Teleport costs (mana-based rules) + resource system hooks
  • Items subsystem + drop tables (config-driven)
  • Holograms support (config-driven)
  • Vault economy integration
  • Core player data model and async-safe persistence
  • Land claims and protections
  • Towns / Territories v1 (membership, ranks, treasury, claim rules, protection)
  • Unified XP/Level API bridge (skill-aware XP service + skill registry integration)

Partially Implemented

  • Auction filters and search UX
  • Shop editor quality-of-life actions (rename/description visibility and prompt safeguards)
  • Market dashboard metrics and visualization
  • Dynamic pricing profiles and tuning (framework present; typically disabled by config)
  • Report management tooling and staff flows
  • Audit logging expansion and views
  • Region integrations beyond basic rectangles
  • Offline delivery as a generalized service (currently auction-first)
  • Placeholder expansion wiring (PlaceholderAPI present; expansion coverage still being built)
  • HUD/UI polish for resources (mode support exists; layout/stacking refinement ongoing)
  • Claims/Towns feature expansion (GUI flows, upgrades/perks, diplomacy layers)
  • Balancing passes for progression values (XP tables, pacing, costs)

Missing / Not Yet Implemented

  • Quest framework and achievements
  • Classes and specializations
  • Ability systems and advanced cooldown tiers
  • Custom items with stats, rarities, and affixes
  • Unified content-authoring tools (NPCs, quest chains, world event scripting)
  • Villages module (expanded social system beyond territories)
  • Exploration systems: elites, bosses, dungeons, and world events
  • Full persistence for advanced combat states and temporary buffs across restarts
  • MySQL/PostgreSQL support
  • Multi-server sync and proxy compatibility

Development Roadmap

The forge burns bright. Below is the planned order of development for RunestoneMMO’s major systems. Each phase builds upon the last, expanding the world into a full RPG experience while keeping RunestoneCore stable and modular.

Phase 1 — Core Completion

  • Finish auction filters and search
  • Complete shop editor UX and safeguards (rename, description, prompt safety)
  • Finalize market dashboard metrics and displays
  • Harden maintenance mode and staff alerts
  • Improve report management flows
  • Expand audit logging coverage and views
  • Refine region handling as a shared RegionService
  • Introduce a generalized DeliveryService for offline rewards
  • Polish resource HUD and costs (mana/stamina UX and configuration)
  • Expand PlaceholderAPI coverage for economy/combat/market stats

Phase 2 — XP & Progression Core

  • Unify XP/Level logic behind a single API (skill-aware XP service + registry)
  • Shared XP gain and level-up events for all modules
  • Scoreboard and placeholder integration for levels and XP
  • Skill display consistency across commands and UI
  • Balancing + tuning pass for XP pacing and progression feel

Phase 3 — Combat & Attributes Polish

  • Refine damage formulas and scaling rules
  • Implement temporary attributes and buff/debuff durations
  • Improve combat feedback (actionbars, indicators, sounds)
  • De-duplicate legacy combat paths and finalize the authoritative route

Phase 4 — Content Systems

  • Quest framework (objectives, rewards, conditions)
  • Achievements and title unlocks
  • Loot table expansion + content-driven drop routing
  • World events framework (timers, triggers, announcements)
  • Admin content tooling (authoring flow and safety)

Phase 5 — World & Adventure

  • Elites, bosses, and dungeon encounters
  • Exploration expansions (biome discovery rewards, landmarks, points of interest)
  • Seasonal cycles and realm events
  • Progression loops tied into economy and world content

Phase 6 — Scaling & Persistence

  • MySQL/PostgreSQL backends
  • Multi-server sync and proxy-aware compatibility
  • Performance hardening and profiling targets
  • Long-term persistence for advanced combat/progression states
