Current State of Development

RunestoneCore is a powerful engine with many systems already implemented — but the true RPG layer (combat, leveling, skills, quests, guilds) is still being forged. Below is the current state of the project based on the latest internal audit.

Fully Implemented

  • Multi‑currency economy
  • Banking system
  • Currency exchange
  • Player shops (basic)
  • Auctions (basic)
  • Moderation suite
  • Chat control
  • Staff chat
  • Jailing system
  • Scoreboards
  • Menu framework
  • Cooldown system
  • Mob health bars
  • World rules
  • Sleep system

Partially Implemented

  • Auction filters & search
  • Shop editor actions
  • Market dashboard metrics
  • Report management
  • Audit logging
  • Teleport command
  • Inventory view tools
  • Attribute system (infrastructure only)

Missing / Not Yet Implemented

  • Combat engine
  • Damage math & attribute integration
  • Leveling system
  • Skills & abilities
  • Classes
  • Quests
  • Guilds / clans
  • Parties
  • Chat channels
  • Custom items & item stats
  • Monster scaling
  • Professions
  • Persistent cooldowns
  • Persistent attributes
  • MySQL/PostgreSQL support

Development Roadmap

The forge burns bright. Below is the planned order of development for RunestoneMMO’s major systems. Each phase builds upon the last, expanding the world into a full RPG experience.

Phase 1 — Core Completion

  • Finish auction filters & search
  • Complete shop editor UI
  • Finalize market dashboard metrics
  • Implement maintenance mode
  • Improve report management
  • Expand audit logging
  • Persistence improvements

Phase 2 — RPG Engine

  • Combat engine (damage formulas)
  • Attribute-driven combat
  • Critical hits, armor, resistances
  • Combat UI (damage indicators)
  • Monster scaling

Phase 3 — Progression Systems

  • Leveling system
  • Skill points
  • Classes & specializations
  • Abilities & cooldowns
  • Professions (mining, smithing, etc.)

Phase 4 — Content Systems

  • Quest framework
  • Achievements & titles
  • Custom items & rarities
  • Item stat system
  • Loot tables & world drops

Phase 5 — Social Systems

  • Parties
  • Guilds / clans
  • Guild ranks & permissions
  • Guild chat
  • Chat channels (local, trade, global)

Phase 6 — Towns Module

  • Town creation
  • Chunk claiming
  • Town protections
  • Town treasury
  • Town upgrades
  • Town diplomacy

Phase 7 — World Content

  • Exploration systems
  • Elites & bosses
  • Dungeons
  • World events
  • Seasonal resets

Phase 8 — Network & Scaling

  • MySQL/PostgreSQL support
  • Multi-server sync
  • Proxy compatibility
  • Async data pipelines
