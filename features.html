Server Features

A full look at the systems, plugins, and current feature structure behind Runestone Vale.

Runestone Vale is built as a custom fantasy MMORPG-style realm where the server is shaped through connected systems, not a random stack of unrelated survival plugins. The goal is to make the world feel cohesive — a realm where progression, land, utility, worldbuilding, presentation, and player identity all support one another.

At the center of the realm are two main project pillars: RunestoneCore and RunestoneSkills. Around them is a broader support stack that strengthens access, permissions, custom content, NPC support, world control, cross-version compatibility, and Bedrock connection support.

The Main Feature Pillars of the Realm

The strongest identity of Runestone Vale comes from its custom systems. These are the main foundations that give the world its long-term direction and make it feel like more than a generic server.

RunestoneCore

RunestoneCore is the foundational engine of the realm. It handles the deeper shared systems that make a custom world function as one: economy direction, menus, scoreboards, homes and teleport utility, claims, towns, command structure, and broader world framework.

It is the system foundation beneath the server — the layer that helps make Runestone Vale feel structured, stable, and expandable.

RunestoneSkills

RunestoneSkills is the professions and progression pillar of the realm. It gives the world Mining, Smithing, Forestry, Fishing, Exploration, profession XP, custom resource direction, custom ore flow, and skill-based long-term player growth.

It is what helps the realm feel worked, lived in, and shaped by player effort instead of just occupied.

Feature Breakdown by Plugin

RunestoneCore

  • Economy foundation: shared financial direction and the base structure for realm value flow
  • Scoreboards and UI: player-facing display systems and cleaner server presentation
  • Homes and teleport utility: travel-oriented player convenience and realm usability
  • Claims: land protection structure supporting persistent world building
  • Towns and territory systems: broader grouped land direction and settlement potential
  • Moderation and utility commands: deeper command-side world control and administration support
  • Menus and GUI systems: cleaner shared interaction systems across the realm
  • Mob health bars: stronger player feedback and world presentation

RunestoneSkills

  • Mining: profession-based gathering and progression tied to resources
  • Smithing: profession-based crafting and refinement progression
  • Forestry: tree-focused gathering and woodcutting progression
  • Fishing: profession-based catch progression for non-combat advancement
  • Exploration: progression tied to travel, movement, and world interaction
  • Custom ore generation: expanded material identity beyond vanilla-only resources
  • Custom resource items: custom progression materials helping define the realm economy and crafting flow
  • Tree Capitator: forestry-oriented quality-of-life and gathering flow support
  • Profession XP: long-term skill progression across the active profession set

Oraxen

  • Custom items: unique realm items beyond standard vanilla limits
  • Custom blocks: expanded building, resource, and world identity options
  • Resource pack integration: stronger visual identity for the server
  • Custom material support: helps Runestone Vale feel distinct from standard survival presentation

Citizens

  • NPC support: foundation for world characters and interactive entities
  • Future quest/world interaction potential: supports a deeper living-world direction
  • Realm atmosphere: helps the world move toward more than purely player-only static spaces

WorldGuard

  • Protected regions: controlled areas for builds, events, safe zones, and curated spaces
  • World control: stronger environmental management and rule enforcement in selected areas
  • Realm safety: helps protect important spaces and keep the server more intentional

WorldEdit

  • Large-scale building support: world shaping and structural creation at a higher level
  • Terrain editing: faster world development and stronger custom environment control
  • Realm construction workflow: supports the handcrafted identity of Runestone Vale

Geyser-Spigot

  • Bedrock connection support: allows Bedrock players to access the realm
  • Cross-platform reach: broadens who can experience the world
  • Community accessibility: helps bring more players into the same realm

Floodgate

  • Bedrock login support: smoother Bedrock-side access flow
  • Cross-platform usability: reduces friction for Bedrock players joining the server
  • Support layer for Geyser: strengthens the bridge between Java-hosted systems and Bedrock access

LuckPerms

  • Permissions and ranks: role control and access management
  • Administrative structure: helps organize server authority and player access levels
  • Scalable control: supports long-term management as the realm grows

CommandAPI

  • Command structure support: cleaner and more advanced command handling
  • Framework quality: helps support more professional command systems inside the realm
  • Development support: useful for expanding custom server commands cleanly over time

PlaceholderAPI

  • Dynamic display integration: allows server information to appear across UI systems
  • Scoreboard support: strengthens player-facing data presentation
  • Compatibility layer: helps different systems display information more cleanly together

Vault

  • Economy compatibility: support bridge for financial systems and integrations
  • Plugin interoperability: helps the server’s economy structure work with wider supported systems
  • Foundation support: a practical layer beneath economy-related behavior

ViaBackwards / ViaRewinds / ViaVersions

  • Version compatibility support: broader client-version access handling
  • Player accessibility: reduces friction for players on different client versions
  • Realm reach: helps the server remain more accessible across version differences

How These Features Work Together

What makes Runestone Vale different is not just the individual presence of these plugins, but the way they support a larger identity. The custom systems define the realm. The support stack strengthens access, world structure, custom content, and usability.

RunestoneCore gives the world its underlying structure. RunestoneSkills gives the world its profession-driven progression and gathering identity. Oraxen deepens presentation and material uniqueness. Citizens strengthens future and current NPC/world interaction potential. WorldEdit and WorldGuard help shape and protect the realm. Geyser and Floodgate open the world to Bedrock players. LuckPerms, CommandAPI, PlaceholderAPI, Vault, and the Via suite help keep the entire project cleaner, more accessible, and more maintainable.

In other words: the server is built so the plugins support the realm, not so the realm feels trapped under the plugins.

Current Realm Identity

Runestone Vale is not trying to be every kind of server at once. It is being shaped into a fantasy MMORPG-style world with mountain-hall atmosphere, dwarven and rune-forged visual influence, structured progression, protected land, custom items and resources, profession-based identity, and a stronger sense of place than a generic survival world.
