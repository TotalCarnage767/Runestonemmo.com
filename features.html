Server Features

What makes Runestone Vale more than a basic survival server.

Runestone Vale is built as a custom fantasy MMORPG-style realm where progression, economy, exploration, professions, custom content, and server systems are designed to work together as one world.

Instead of feeling like a loose collection of plugins and commands, the goal is to make the realm feel cohesive — a place where systems support the world, and the world supports long-term player progression.

Live Gameplay Direction

The goal of Runestone Vale is to create a world where different playstyles can all matter. Gatherers, builders, traders, explorers, and long-term players should all feel like they have a place in the realm.

That means progression is not only about combat or raw item grind. It is also about professions, discovery, economy, land, crafting, reputation, and the growing role a player takes within the world itself.

Every update is aimed at making those systems feel more connected, more immersive, and more meaningful.

Server Systems in the Realm

Custom Core Systems

  • RunestoneCore powers the deeper framework of the realm
  • Economy, player systems, travel, claims, menus, scoreboards, and shared logic live here
  • Acts as the central engine behind the wider RunestoneMMO ecosystem

Progression Systems

  • RunestoneSkills powers professions and long-term advancement
  • Designed to support gathering, crafting, progression, and world interaction
  • Built to expand as the realm grows

Custom Content

  • Oraxen supports custom items, blocks, textures, and resource-pack content
  • Helps give the realm its own visual identity and custom resource progression
  • Supports the custom fantasy feel of the world

Protection & Building Support

  • WorldGuard helps protect key areas and define important regions
  • WorldEdit supports world shaping, terrain work, and large-scale building
  • Together they help maintain and forge the realm itself

NPC & World Interaction

  • Citizens supports NPC-driven interaction and world presentation
  • Works alongside the growing NPC / Citizens bridge direction in the project
  • Supports future realm features such as guides, interaction, and quest content

Access & Permissions

  • LuckPerms handles permissions and server role structure
  • Vault provides compatibility for economy and service hooks
  • These systems help hold the administrative structure together cleanly

Cross-Platform & Compatibility Support

Runestone Vale is being built to be more accessible and more stable across different client situations, while still preserving the core Java server experience.

Java & Bedrock Access

  • Geyser-Spigot helps open the realm to Bedrock players
  • Floodgate supports Bedrock-friendly account bridging
  • This helps broaden access to the world without abandoning the core Paper platform

Version Compatibility

  • ViaVersion, ViaBackwards, and ViaRewind help support version flexibility
  • These systems improve compatibility for players joining from different client versions
  • Useful for keeping the community more accessible while development continues

Presentation Hooks

  • PlaceholderAPI supports dynamic placeholder-driven display content
  • Helps feed scoreboards, UI layers, and server-facing presentation systems
  • Supports a more polished and informative player experience
