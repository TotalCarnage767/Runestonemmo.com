What RunestoneCore Is

RunestoneCore is the shared platform layer for the RunestoneMMO project. It is designed to give servers a strong, extensible base for persistent gameplay systems without forcing every future feature into one monolithic plugin.

The goal is simple: one professional core that can support economy, progression, moderation, travel, UI, combat scaffolding, and future modules cleanly. RunestoneSkills, future quest systems, content systems, and other expansions are built on top of this spine — not mixed directly into it.

Core Systems

Configuration & Lifecycle

  • Multi-file YAML configuration for core systems and subsystem tuning
  • Central routed access through configuration managers
  • Deterministic bootstrap and shutdown flow for subsystem safety
  • Central task and listener registration to reduce duplication and orphaned behavior
  • Adventure / MiniMessage-driven output and formatting support

Player Data & Profiles

  • Persistent player profiles with preferences, toggles, titles, and progression state
  • Combat XP and related progression support
  • Multi-currency wallet and bank storage
  • Daily reward tracking and completion state
  • Async-safe load / save flow with cache-aware handling

Economy Platform

  • Multi-currency economy design
  • Wallet balances, bank balances, and server treasury support
  • Tax routing and configurable economy sinks
  • Exchange support between currencies
  • Ledger / history systems for tracking transactions
  • Vault compatibility for wider plugin integration

Auctions, Shops & Market Flow

  • Auction house support with listing and expiry flow
  • Player shop systems and shop creation flow
  • Shop editing and description support
  • Market dashboard style overview systems
  • Integrated economy and logging behavior

Daily Rewards

  • Daily reward claims and status tracking
  • Calendar / streak oriented reward flow
  • Support for currency, items, and command rewards
  • Designed for configurable progression-oriented reward tables

Combat & Attributes

  • Combat XP support and combat progression hooks
  • Attribute scaffolding for attack, defense, toughness, crits, and scaling
  • Mob attribute application support
  • Combat-facing systems designed to support future RPG expansion

Claims & Protection

  • Personal land claim system
  • Protection logic for building, containers, and related interactions
  • Trust / flag style territory control
  • Protection-aware world interaction rules

Towns & Territories

  • Town creation and structured settlement support
  • Rank, membership, treasury, and claim-based territory rules
  • Expansion logic designed around controlled chunk growth
  • Integrated direction for future settlement gameplay

Scoreboards & Menus

  • Config-driven scoreboards with multiple layouts
  • Conditional displays for context-specific UI behavior
  • Menu framework for structured GUI systems
  • Active refinement toward cleaner persistence and display flow

Resources & HUD Systems

  • Mana and stamina resource support
  • BossBar / ActionBar oriented HUD channels
  • Teleport mana-cost integration and resource hooks
  • Designed to support wider RPG resource systems later

Travel & Teleportation

  • Home system support
  • Teleport request flow
  • Spawn return flow
  • Back-location support
  • Utility travel systems for both players and staff

Moderation & World Rules

  • Staff moderation utilities and punishment flow
  • Chat control and server-side management tools
  • Maintenance mode and broadcast systems
  • World-rule control, gameplay toggles, and administrative enforcement

Tasks, Logging & APIs

  • Central task and cleanup patterns for scheduled systems
  • Logging support for economy and staff actions
  • Public-facing APIs and integration points for future modules
  • Built to serve as the long-term backbone of the wider project

What This Page Does Not Cover

RunestoneCore is the backbone of the project, but not every gameplay system belongs on this page. Dedicated profession and progression content belongs to the RunestoneSkills module and its own page.

Not the Skills Module

  • Mining, Forestry, Fishing, Smithing, Foraging, and other skill systems are documented separately
  • Profession-specific balance and progression loops belong on the RunestoneSkills page

Not the Full Content Layer

  • Quest frameworks, large content systems, and world-event layers are still future expansion areas
  • RunestoneCore is meant to power those systems, not replace their dedicated modules

Not the Final Network Scale Layer

  • The platform direction is long-term and still expanding
  • Broader scaling, deeper persistence options, and future network-ready layers remain part of the roadmap
